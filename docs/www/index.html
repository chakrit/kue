Kue

A strongly-typed reimplementation of the CUE v0.15 language in Lean 4 — correctness-first.

What works

Evaluator + lattice Total meet/join over the full Value domain: primitives, kinds, bounds, regex, struct/list shapes. Disjunctions with full default-mark algebra. 
port: int & >0 & <=65535
port: 8080  // 8080
Closures / cross-package def-meet Value.closure carries the capture frame so an imported def unifies at the use site before cross-frame self/sibling refs resolve. Deep self-ref detection; multi-level embed chains. 
import "ex.com/pkg"
web: pkg.#Def & {name: "web"}
Comprehensions Struct and list comprehensions, including if guards, nested / multi / zero-yield, plain+comp interleave, and scalar struct-embedding collapse. 
out: [for x in [1, 2, 3] {x * 2}]  // [2, 4, 6]
Disjunction defaults under embedding Use-site narrowing distributes into every arm of an embedded default disjunction, pruning dead arms. 
x: (*"a" | "b") & ("b" | "c")  // "b"
Fuel-saturation perf Eval count is flat across fuel (bracketed monotonic truncation counter; saturated results go fuel-free). Frame-id sharing + force-memo, partial.
Builtins base64, json.Marshal, yaml.Marshal, and strings / list / math namespaces. Multiline strings. 
import "encoding/json"
out: json.Marshal({a: 1})  // "{\"a\":1}"
Imports / modules cue.mod discovery, in-module and cross-module resolution by longest module-path prefix, multi-file package merge, transitive loads, package-dir entry. IO confined to the loader; eval stays pure. 
import "ex.com/pkg"
out: pkg.#Def & {name: "x"}
CLI kue eval and kue export [--out yaml|json] [file|dir] over stdin or arg, with clean missing-file diagnostics and exit codes.

Philosophy

Kue targets CUE v0.15 semantics, done correctly. Where the official cue binary is buggy, Kue implements the correct behavior — the compatibility target is the language as specified, not bug-for-bug parity.

Correctness comes before performance: a latent unsound result is a violation even with no failing fixture, while a perf miss is acceptable. Lean 4 is the chosen language to push these invariants into the type system — total functions and precise types over ad hoc runtime checks. See the compatibility target and correctness over performance.
