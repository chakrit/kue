Kue

A strongly-typed reimplementation of the CUE v0.15 language in Lean 4 — correctness-first, grounded on the spec, not on the cue binary.

Now cert-manager exports as a fast, content-identical drop-in. The spec-conformance re-audit is complete — every behavior regraded against the CUE spec, not byte-matched to a fallible binary — and its high-severity fixes (RE2 regex, structural cycles, guard handling, closedness) have all landed.

The reframe

The single most important thing to know about where Kue is right now.

Kue exists because cue is buggy — yet the build had quietly drifted into treating byte-identical-to-cue as the definition of "correct." That gate is structurally bug-replicating: every time Kue differed, the reflex was to match cue, which hides the very divergences Kue is meant to fix. We flipped it. The authority is now the CUE language spec; where the spec is silent, lattice first principles; the cue binary is a fallible cross-check, never the gold standard. A full six-area re-audit followed — and found real, spec-grounded bugs the old gate had masked (a non-RE2 regex engine, missing structural-cycle detection, a missing stdlib import, a guard that silently swallowed errors, a closed struct that could be re-opened) while vindicating the bulk of the engine as genuinely correct. Every one of those high-severity findings has since been fixed.

The spec-conformance re-audit

Six behavioral areas regraded against the spec + lattice. Each behavior gets one verdict. Tracked in spec-conformance-audit.md.

CONFORMS Spec speaks, Kue matches it. The numeric/bounds/division/decimal core, the lattice meet, default-mark algebra, the frame model — all verified correct, not merely cue-matched.
KUE-VIOLATES Spec speaks, Kue is wrong (often from matching a cue bug). ~9 found — the regex engine, structural cycles, a closedness soundness hole, the comprehension guards — and every high-severity one has since been fixed.
CUE-BUG / KUE-CORRECT Spec speaks, cue is wrong, Kue follows the spec. Recorded in cue-divergences.md — deliberate, defensible disagreement with the binary.
SPEC-SILENT The spec doesn't say. Kue derives the answer from lattice principles and records the gap in cue-spec-gaps.md — even when it happens to match cue (matching an artifact is lower-confidence).

What works

Evaluator + lattice Total meet/join over the full Value domain: primitives, kinds, bounds, regex, struct/list shapes. Disjunctions with full default-mark algebra — all re-audited as spec-correct. 
port: int & >0 & <=65535
port: 8080  // 8080
One unified struct (B2) Five struct constructors + a 12-arm meet matrix collapsed into one normalized struct with orthogonal openness/tail/pattern axes — erasing a whole class of missing-meet-arm bugs by construction. 
{[string]: int} & {a: 5, ...}  // {a: 5}, open
Closedness / definitions Closed #Defs, ... openness, recursive closing, and import-binding laziness — hardened by the spec-first fixes (SC-1, SC-1c, SC-1d): a closed struct now stays closed through a pattern meet. 
#C: {a: int}
out: #C & {z: 9}  // z: field not allowed
Comprehensions Struct and list comprehensions, if guards (a bottom guard now propagates rather than vanishing — D#1a), nested / multi / zero-yield, embedded narrowing. 
out: [for x in [1, 2, 3] {x * 2}]  // [2, 4, 6]
Unified frame coordinates (B7) The de Bruijn clause-depth rule (for pushes a frame, if doesn't) lives in one shared descendClauses fold with agreement theorems — so a whole recurring class of frame-depth bugs is now a build failure.
Perf: O(N²) → O(N) memo (item 7) The cache keyed on a shallow constructor tag, collapsing every value into one hash bucket. A bounded-depth structural digest fixed it — provably sound (hash-only; equality untouched). cert-manager 119s → ~30s.
Builtins base64, json.Marshal, yaml.Marshal, strings/list/math, and now regexp.Match (F-1). Numeric tower + decimal verified spec-exact. 
import "encoding/json"
out: json.Marshal({a: 1})  // "{\"a\":1}"
Imports / modules cue.mod discovery, in-module + cross-module resolution by longest module-path prefix, multi-file merge, transitive loads, package-dir entry. IO confined to the loader; eval stays pure. 
import "ex.com/pkg"
out: pkg.#Def & {name: "x"}

Spec-first fix backlog

The re-audit's findings, ranked. Contained high-confidence fixes land before the large rewrites to keep main shippable. See the audit tracker for the full list.

All HIGH-severity findings — RX-1 (RE2 regex), D#2 (structural cycles), Gap-2b, SC-1/2 (closedness), D#1a-c (guards), F-1/2/3 Landed HIGH · spec-grounded

The regex engine is now a total RE2-equivalent AST → NFA matcher (\b, lazy quantifiers, in-class set-complement); recursive definitions get a structural cycle error and recursive-default termination; a closed struct stays closed through a pattern meet and through instantiation (diverging from a cue bug, on purpose); comprehension guards defer / type-error / propagate correctly; import "regexp", self-module @vN strip, and qualified import paths all resolve. Each justified by a spec citation, cross-checked (not gated) against cue.

MED tail — D#3 (let clauses), BI-2 (math.Pow / list.Sort) Landed MEDIUM

let clauses in comprehensions (the de Bruijn frame model extended, B7-vindicated); math.Pow's exact integer-exponent domain and list.Sort/SortStable with a per-pair-evaluated comparator. The numeric residual (math.Sqrt, fractional Pow — needs an IEEE/Float model) is deferred: Kue bottoms rather than ship a wrong value.

BI-1 — Unicode case folding (data-spike first) Next MEDIUM · envelope risk

strings.ToUpper/ToLower are ASCII-only; cue is full-Unicode, so non-ASCII gives wrong answers. The slice must FIRST decide the data approach — a checked-in vendored case-folding table (no fetch) vs scoped coverage — before any code, since fetching external data into the repo is an envelope boundary.

argocd Bug2-5 — let-buried disjunction-arm narrowing Parked stress-test finding · not on the critical path

The one remaining argocd blocker: a sibling let narrowed by a co-embedding def's static field, where the embed resolves on a disjunction arm so the narrowing arrives too late. Real, well-understood, but needs app-specific narrowing — so it is parked as a stress-test finding and resolves as the general semantics mature, rather than being chased onto the critical path.

Hardening + cleanups Backlog LOW/MED · opportunistic, gated behind correctness

truncate-primitive (make the fuel-truncation bump unskippable by construction — HIGH soundness hardening), the periodic test/fixture-org and plan-hygiene passes, SC-3 display-residual + the spec-gap ratifications, and the walker/normalizer dedups (AD4-1 comprehension walkers and A-EN3 frame folds — both DONE; DRY-1 let-walkers ruled out, no shared combinator; AD2-1 disjunction normalizers remaining) — distinct mechanisms, only the ones that genuinely share structure unified. Philosophy

Kue targets CUE v0.15 semantics, done correctly — the compatibility target is the language as specified, not bug-for-bug parity with the reference binary. cue is buggy; replicating it is the wrong default.

So every behavior is judged against an authority hierarchy:

  • 1 · The CUE spec — authoritative where it speaks; match it even against the binary.
  • 2 · Lattice first principles — where the spec is silent (often). Derive the mathematically-correct behavior. This is Kue's reason to exist.
  • 3 · The cue binary — a fallible cross-check. Never the gate.

Correctness comes before performance: a latent unsound result is a violation even with no failing fixture, while a perf miss is acceptable. Lean 4 pushes these invariants into the type system — total functions and precise types over ad hoc runtime checks. See the compatibility target, correctness over performance, and the spec-conformance re-audit.
