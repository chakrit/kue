# Fixture-as-oracle audit — 2026-06-14

Discharges the consequence in
[`../decisions/2026-06-14-cue-compatibility-target.md`](../decisions/2026-06-14-cue-compatibility-target.md):
the `testdata/cue/*.expected` files were generated by cross-checking against the official
`cue eval` binary, so they could encode v0.15 binary bugs rather than the intended
semantics Kue targets. This audit checked them against the spec, not the binary.

## Method

All 82 fixture pairs audited in one read-only pass, split into four semantic domains
(fanned out to parallel subagents), each judging `.expected` against intended CUE v0.15
spec semantics (cuelang.org spec + RE2 for regex), conservatively — flag only
high-confidence wrong outputs:

- **Arithmetic / expressions** (16) — additive, multiplicative, division, integer-keyword
  `div`/`mod`/`quo`/`rem`, unary, equality, ordering, numeric comparison, logical,
  non-decimal, literals, suffixes.
- **Structs / lists / manifest** (28) — closedness, field classes, unification, list
  unification, export filtering, default selection, kind meets.
- **Disjunctions / bounds / references / cycles** (16) — default markers, subsumed-disjunct
  elimination, bounds unification, reference resolution, structural cycles, `let`.
- **Regex / patterns / builtins** (23) — pattern constraints, RE2 matching, `=~`/`!~`,
  `len`/`and`/`or`/`div`/`mod`/`quo`/`rem`.

## Result

**No high-confidence suspect fixtures.** Every `.expected` encodes correct intended CUE
v0.15 semantics, including the spots most prone to reference-binary bugs:

- Decimal (apd) arithmetic — `0.1 + 0.2 = 0.3` exactly, `1/3` to 34 digits.
- `div`/`mod` Euclidean (non-negative `mod`), `quo`/`rem` truncating — signs correct.
- Structural cycles — `x: x & >=0` → `>=0`; pure ref cycles collapse to `_`.
- Disjunction defaults and subsumed-disjunct elimination (`number | 1` → `number`).
- Closedness rejections and RE2 match results.

Kue's deliberate, internally-consistent output conventions (inline `field: _|_` for
conflicts vs the binary's top-level hard error; hidden fields shown in eval output as with
`cue eval -a`) are presentation, not semantic divergence, and are applied uniformly.

## Lone open item (low confidence, already documented)

`number_literals`: `1.25e3` renders as `1.25e+3` (exponent form retained, sign normalized)
rather than expanding to `1250`. Two auditors independently flagged this as
format-/version-sensitive. Already a documented boundary in
[`../spec/compat-assumptions.md`](../spec/compat-assumptions.md) §Numeric literals ("Kue
does not yet canonicalize all exponent arithmetic the way `cue eval` does"). Not a bug;
left as-is pending a future canonicalization slice.

## Takeaway

The fixture suite is trustworthy as an oracle for the implemented surface. Future TDD
slices can lean on it without inheriting reference-binary bugs. No fixture changes made.
