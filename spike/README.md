# Spike: Lean engine embedded in a Go binary (cgo)

A feasibility spike proving Kue can ship as a **single Go binary** that owns `main` and
calls the **real Lean evaluator in-process** via cgo — no `kue` subprocess. Verdict:
**COMMIT**. Full write-up:
[`docs/decisions/2026-06-25-lean-engine-embedded-in-go-via-cgo.md`](../docs/decisions/2026-06-25-lean-engine-embedded-in-go-via-cgo.md).

This is exploratory scaffolding, not production code. It is isolated from `lake build kue`
and the release scripts.

## What it does

A single cgo binary evaluates CUE through Kue's actual engine and prints:

- `x: 1.5 + 2.5` → `x: 4.0` (exercises `Decimal`/GMP bignum)
- `a: 1` / `b: a + 1` → `a: 1` / `b: 2` (exercises reference resolution)

## Files

| File                       | Role                                                          |
| -------------------------- | ------------------------------------------------------------- |
| `Spike.lean`               | `@[export kue_spike_eval]` wrapping `Kue.evalSourceToString`. |
| `shim.c` / `shim.h`        | C shim: Lean init + Lean-object↔`char*` marshaling.           |
| `main.go`                  | Go `main`; cgo-calls the shim, asserts the results.           |
| `build.sh`                 | macOS/host build: lake → archive → cgo link → run.            |
| `Dockerfile.linux-spike`   | Linux build: same, in a container; RUN runs the spike.        |
| `cgo_flags.go`             | Generated by `build.sh` (platform LDFLAGS). Not committed.    |

## Build & run

### macOS / host (needs Go + the elan toolchain)

```sh
bash spike/build.sh
```

Builds and runs; expect `ALL CASES PASSED`.

### Linux x86_64 (in a container; QEMU on an arm64 host — slow)

```sh
docker buildx build \
  --platform linux/amd64 \
  --build-arg LEAN_TOOLCHAIN=v4.29.1 \
  --file spike/Dockerfile.linux-spike \
  --tag kue-linux-spike:amd64 --load .
```

The image's final `RUN` runs the spike; a green build is the verdict.

## The link line (the crux)

Derived empirically from `leanc --print-ldflags`:

```
libkuespike.a -L$(lean --print-prefix)/lib/lean -L$(lean --print-prefix)/lib \
  -lleancpp -lleanrt -lInit -lStd -lLean -lgmp -luv <-lc++ | -lstdc++> -lm
```

See the decision note for the init sequence, the owned-vs-borrowed double-free trap, and the
other sharp edges.
