Kue

A strongly-typed reimplementation of the CUE v0.15 language in Lean 4 — correctness-first, grounded on the spec, not on the cue binary.

Now cert-manager exports as a fast, content-identical drop-in for cue (~11.7s) — the live in-gate canary. The broader prod9 corpus (lem / n8n / x9 / typesense) drove a layer-by-layer eval-conformance campaign: each blocking construct was lifted into a self-contained failing fixture and fixed with a general, spec-grounded change — L1–L5 complete (list-embedding Self.#hidden, default-disjunction interpolation, let-delivered list carriers, closedness through embedded disjunctions, and open-tail-operand closedness). The spec-conformance re-audit is complete and every high-severity finding has landed: the RE2 regex engine, structural cycles, the full closedness family, the dyn-field and residual-through-meet families, Unicode case folding, full-domain math.Pow/Sqrt. Imports are now file-scoped and let/alias shadow is rejected at load. Latest cut: v0.1.0-alpha.20260623 · Lean 4.31.

The reframe

The single most important thing to know about where Kue is right now.

Kue exists because cue is buggy — yet the build had quietly drifted into treating byte-identical-to-cue as the definition of "correct." That gate is structurally bug-replicating: every time Kue differed, the reflex was to match cue, which hides the very divergences Kue is meant to fix. We flipped it. The authority is now the CUE language spec; where the spec is silent, lattice first principles; the cue binary is a fallible cross-check, never the gold standard. A full six-area re-audit followed — and found real, spec-grounded bugs the old gate had masked (a non-RE2 regex engine, missing structural-cycle detection, a missing stdlib import, a guard that silently swallowed errors, a closed struct that could be re-opened) while vindicating the bulk of the engine as genuinely correct. Every one of those high-severity findings has since been fixed.

The spec-conformance re-audit

Six behavioral areas regraded against the spec + lattice. Each behavior gets one verdict. Tracked in spec-conformance-audit.md.

CONFORMS Spec speaks, Kue matches it. The numeric/bounds/division/decimal core, the lattice meet, default-mark algebra, the frame model — all verified correct, not merely cue-matched.
KUE-VIOLATES Spec speaks, Kue is wrong (often from matching a cue bug). ~9 found — the regex engine, structural cycles, a closedness soundness hole, the comprehension guards — and every high-severity one has since been fixed.
CUE-BUG / KUE-CORRECT Spec speaks, cue is wrong, Kue follows the spec. Recorded in cue-divergences.md — deliberate, defensible disagreement with the binary.
SPEC-SILENT The spec doesn't say. Kue derives the answer from lattice principles and records the gap in cue-spec-gaps.md — even when it happens to match cue (matching an artifact is lower-confidence).

What works

Evaluator + lattice Total meet/join over the full Value domain: primitives, kinds, bounds, regex, struct/list shapes. Disjunctions with full default-mark algebra — all re-audited as spec-correct. Structural-cycle detection (a recursive #L:{n,next:#L} errors; #List | *null terminates on the default). The disjunction-prune predicate containsBottom is total — no fuel cap — so a bottom at any depth is always found. 
port: int & >0 & <=65535
port: 8080  // 8080
One unified struct (B2) Five struct constructors + a 12-arm meet matrix collapsed into one normalized struct with orthogonal openness/tail/pattern axes — erasing a whole class of missing-meet-arm bugs by construction. 
{[string]: int} & {a: 5, ...}  // {a: 5}, open
Closedness / definitions Closed #Defs, ... openness, recursive closing. Closedness is carried as closedClauses — one {fieldLabels, patterns} per closed conjunct — so admittance is a per-conjunct intersection (a field survives iff every clause admits it), not a flat union. The whole family is closed (SC-1…SC-1e, SC-2, EMBED-CLOSE-1): a closed struct stays closed through a pattern meet, a closed×open-… meet, and instantiation. 
#C: {a: int}
out: #C & {z: 9}  // z: field not allowed
Comprehensions Struct and list comprehensions with if guards (a bottom guard now propagates rather than vanishing — D#1a) and let clauses (D#3), nested / multi / zero-yield, embedded narrowing. A non-concrete comprehension residual is correctly HELD and survives a meet against a struct rather than collapsing (the MEET-RESID-1 family), guarded by an illegal-states-unrepresentable invariant. 
out: [for x in [1, 2, 3] {x * 2}]  // [2, 4, 6]
Unified frame coordinates (B7) The de Bruijn clause-depth rule (for pushes a frame, if doesn't) lives in one shared descendClauses fold with agreement theorems — so a whole recurring class of frame-depth bugs is now a build failure.
Perf: O(N²) → O(N) memo (item 7 + perf #7) The cache keyed on a shallow constructor tag, collapsing every value into one hash bucket. A bounded-depth structural digest fixed it — provably sound (hash-only; equality untouched). Perf #7 then added an env-independent-leaf fast path. cert-manager 119s → ~11.7s; argocd ~50.3s.
Builtins base64, json.Marshal, yaml.Marshal, regexp.Match, and the strings/list/math namespaces. Now full-Unicode strings.ToUpper/ToLower (BI-1, an oracle-generated case-fold table) and full-real-domain math.Pow/Sqrt in exact decimal (BI-2 — Float correctly avoided) + list.Sort/SortStable. Numeric tower + decimal verified spec-exact. 
import "encoding/json"
out: json.Marshal({a: 1})  // "{\"a\":1}"
Imports / modules cue.mod discovery, in-module + cross-module resolution by longest module-path prefix, multi-file merge, transitive loads, package-dir entry, and qualified import paths "location:identifier" (F-3). IO confined to the loader; eval stays pure. 
import "ex.com/pkg"
out: pkg.#Def & {name: "x"}

perf #7 — share evaluations across frame scopes Won't-fix measurement-driven rejection · 2026-06-23

A pure perf concern — no correctness divergence. The profile named a ~175× re-evaluation factor (the same subtree re-evaluated under ~175 distinct frame scopes, and the cache keys on frame identity). Two safe wins shipped (an env-independent-leaf fast path + a dead-cache removal, both value-identical by construction). The designed next step — collapsing evaluations across frame scopes — was then measured and rejected: a zero-risk content-addressed shadow put the share ceiling at 0.045% / 0.059% (cert-manager / argocd). The re-eval is genuinely distinct content reached under different observable bindings, not redundancy, so collapsing it would be a false share (a wrong value). Correctness comes before performance — no sound frame-sharing reclaims it, so this is WON'T-FIX; the residual wall is the irreducible cost of distinct content.

Philosophy

Kue targets CUE v0.15 semantics, done correctly — the compatibility target is the language as specified, not bug-for-bug parity with the reference binary. cue is buggy; replicating it is the wrong default.

So every behavior is judged against an authority hierarchy:

  • 1 · The CUE spec — authoritative where it speaks; match it even against the binary.
  • 2 · Lattice first principles — where the spec is silent (often). Derive the mathematically-correct behavior. This is Kue's reason to exist.
  • 3 · The cue binary — a fallible cross-check. Never the gate.

Correctness comes before performance: a latent unsound result is a violation even with no failing fixture, while a perf miss is acceptable. Lean 4 pushes these invariants into the type system — total functions and precise types over ad hoc runtime checks. See the compatibility target, correctness over performance, and the spec-conformance re-audit.
